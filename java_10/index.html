2.2 选择器

写法

选择器是一个选择谁（标签）的过程。

基础选择器

标签选择器

标签{属性：值;}

特点：标签选择器定义之后，会将页面所有的元素都执行这个标签样式。

<head>
	<style type="text/css">
        div{
            font-size: 20px;
            color:#000000;
            width:100px;
            height:100px;
            background-color: red;
        }
    </style>
</head>
<body>
<div>我爱北京</div>
<div>你好</div>
</body>

颜色的显示方式

  • 直接写颜色的名称：red\blue\green…

  • 十六进制显示颜色：0-9 a-f #000000; 前2为代表红色，中间2位代表绿色，后边2位代表蓝色。

类选择器（重点）

写法

.自定义类名{属性:值; 属性:值；}

<head> 
	<style type="text/css">
        .box{
            font-size:40px;
            color:red;
            width:200px;
            height:100px;
            background-color: #000033;
        }
        .miss{
            text-indent:2em;
        }
    </style>
</head>

<body>
<div class="box miss">我爱北京</div>
<div class=”box”>天安门</div>
</body>

特点： 谁调用，谁生效。

  • 一个标签可以调用多个类选择器。

  • 多个标签可以调用同一个类选择器。

类选择器命名规则

  • 不能用纯数字或者数字开头来定义类名

  • 不能使用特殊符号或者特殊符号开头（_）来定义类名

  • 不建议使用汉字来定义类名

  • 不推荐使用属性或者属性的值来定义类名

ID选择器

写法

#自定义名称{属性:值;}

特点： 一个ID选择器在一个页面只能调用一次。如果使用2次或者2次以上，不符合w3c规范，JS调用会出问题。

  • 一个标签只能调用一个ID选择器。

  • 一个标签可以同时调用类选择器和ID选择器。

<head>
    <style type="text/css">        
        #box{
           color:blue;
           background-color:red;
        }
        .miss{
             width: 400px;
        }
    </style>
</head>

<body>
<div id="box" class="miss">我爱北京天安门</div>
</body>
通配符选择器
*{属性:值;}

特点：给所有的标签都使用相同的样式。不推荐使用，增加浏览器和服务器负担！！

<head>
	<style type="text/css">        
        *{
           color:blue;

</body>

2.3 文本元素

属性

  • font-size:16px; 文字大小

font: font-style font-weight font-size/line-height font-family;

  • 内嵌式写法

    <style type=”text/css”>
    	样式表写法
    </style>

  • 外链式写法

  • 行内样式表

三种写法特点:

​ 内嵌式写法，样式只作用于当前文件，没有真正实现结构表现分离。

​ 外链式写法，作用范围是当前站点，谁调用谁生效，范围广，真正实现结构表现分离。

​ 行内样式表，作用范围仅限于当前标签，范围小，结构表现混在一起。 （不推荐使用）

  • 可以设置宽高

2.6 链接伪类

定义的时候必须按照以下顺序：其中，a:link{属性:值;} a{属性:值}效果是一样的！

a:link{属性:值;}     // 链接默认状态  

a:visited{属性:值;}  // 链接访问之后的状态 

a:hover{属性:值;}    // 鼠标放到链接上显示的状态  a:active{属性:值;}   链接激活的状态

a:focus{属性:值；}   // 获取焦点

文本修饰

背景属性

  • background-color 背景颜色

  • background-image 背景图片

  • Background-repeat repeat(默认) | no-repeat | repeat-x | repeat-y 背景平铺

  • Background-position left | right | center | top | bottom 背景定位

方位值只写一个的时候，另外一个值默认居中。

写2个方位值的时候，顺序没有要求。

写2个具体值的时候，第一个值代表水平方向，第二个值代表垂直方向。

背景属性连写
background:url("../img/login.gif") no-repeat 30px 50px red;

连写的时候没有顺序要求，url为必写项。

行高

浏览器默认文字大小

  • 浏览器默认文字大小：16px

文档流（标准流）

元素自上而下，自左而右，块元素独占一行，行内元素在一行上显示，碰到父集元素的边框换行。

浮动布局

为什么要有浮动？多个div需要排成一行！float: left | right

特点：

  • 元素浮动之后不占据原来的位置（脱标）
  • 漂浮于原有元素的上一个层面
  • 根据浮动的方向定位于父元素的左上角或右上角。
  • 多个浮动元素会按照流式布局依次排列下去
  • 元素浮动之后转换为行内块元素。

清除浮动

  • 清除浮动不是不用浮动，清除浮动产生的影响。

  • 清除浮动的方法

clear: left | right | both

工作里用的最多的是clear:both;

清除浮动的应用：当嵌套的元素父元素没有设置宽高的时候，子元素浮动会导致边框塌陷。

解决办法：

  1. 额外标签法，在最后一个浮动元素后添加标签,。

  1. 给父集元素使用：overflow:hidden;

  1. 伪元素清除浮动 （推荐使用）
  .clear:after{
       content:'';
       display:block;
       clear:both;
     }

绝对定位
Position:absolute;

特点：

  • 元素使用绝对定位之后不占据原来的位置（脱标）

  • 使用绝对定位之后，转换为行内块。

标签包含规范

  • p标签不能包含div h1等标签。

  • h1可以包含p，div等标签。

  • 行内元素尽量包含行内元素，行内元素不要包含块元素。
